🎓 University Management System

A production-style university management backend focused on async architecture, role-based authorization, transactional consistency, and scalable API design using FastAPI, MySQL, Docker, and Async SQLAlchemy.

FastAPI JWT Auth RBAC MySQL

Core Features

  • JWT Authentication (Signup & Login)
  • Role-Based Access Control (Admin, Faculty)
  • Course Management System
  • Student & Faculty Management
  • Enrollment System
  • Full Test Coverage using Pytest

Frontend Dashboard

  • Signup & Login Interface
  • Create Courses, Students, Faculty
  • Enrollment Management
  • API interaction without Postman

System Operations

Operation Description
Create Course Add new course
Create Student Register student
Create Faculty Add faculty
Enroll Student Assign student to course
Authentication JWT-based login/signup

Architecture

The backend follows a layered async architecture with separated routes, services, queries, and ORM models designed for maintainability, transaction safety, and scalable API operations. 


routes → services → models

auth_service()      → login/signup
course_service()    → course logic
student_service()   → student logic
faculty_service()   → faculty logic
enrollment_service()→ linking system

Security

  • Password hashing with bcrypt
  • JWT token authentication
  • Protected API routes
  • Backend-controlled role permissions

Engineering Challenges

  • Handled async database operations using Async SQLAlchemy + aiomysql
  • Stress-tested authenticated APIs with k6 under high concurrency
  • Implemented role-based route protection with JWT authentication
  • Debugged Docker networking and concurrent database bottlenecks
  • Maintained API stability under mixed authenticated CRUD workloads

Tech Stack

  • Backend: FastAPI + Async Python
  • Database: MySQL

  • Authentication: JWT + bcrypt
  • Testing: Pytest + pytest-asyncio
  • Load Testing: k6
  • Infrastructure: Docker + Docker Compose
